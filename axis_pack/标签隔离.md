## 你关心的“严格单轴”到底屏蔽了什么？

在我给你的 axis mask 版本里，**单轴模式会做到：**

### emotion_only

- ✅ 只让 `emotion` 参与：`emo_text / emo_vector / emo_alpha / 情绪节奏模板`
- ❌ 完全屏蔽 microexpression、speech_rate_anomaly
- ❌ 屏蔽 intention（CRM 情绪提示不参与）
- ❌ 禁用你代码里那条“犹豫 + intention → 不自然停顿”的跨轴规则（避免污染）

### micro_only

- ✅ 只让 `microexpression` 参与：向量注入 & 提示词
- ❌ emotion 屏蔽（作为中性基线）
- ❌ rate 屏蔽（语速相关的切块/停顿/变速不启用）
- ❌ intention 屏蔽

### rate_only

- ✅ 只让 `speech_rate_anomaly` 参与：切块/停顿/变速/（以及你配置里的向量微调）
- ❌ emotion 屏蔽（中性基线）
- ❌ micro 屏蔽
- ❌ intention 屏蔽

> 重点是：这不是“把其它字段强制写成正常”那种数据侧处理；而是在 **合成时** 用 `axis_mode` 决定哪些轴被读取、哪些轴被忽略，从源头避免混入。

## 2）推荐你把 run.sh 改成下面这个“稳版”

（你复制覆盖 axis_pack/run.sh 即可）

```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT="/home/shentong/index-tts-main"
PY="/home/shentong/miniconda3/envs/indextts2/bin/python"

# 必须：让 python 能 import 到仓库里的 indextts
export PYTHONPATH="$ROOT"
cd "$ROOT"

# 可选：固定 GPU（不想固定就删掉这行）
# export CUDA_VISIBLE_DEVICES=0

exec "$PY" axis_pack/export_single_axis.py \
  --input axis_pack/all_conversations.jsonl \
  --speaker_map axis_pack/speaker_map.json \
  --cfg checkpoints/config.yaml \
  --model_dir checkpoints/IndexTTS-2 \
  --configs_dir axis_pack/configs \
  --out_root axis_pack/output_export_by_tags \
  "$@"
```

然后给它可执行权限：

```bash
chmod +x axis_pack/run.sh
```
